Include Scripts

The first step is to include gmapslayer.js or gmapslayer_compiled.js in your document header, after the ArcGIS Server JavaScript API has loaded. You can use the hosted release version if you do not want to download the script. Please note, there is NO need to load Google API in the header. The Google API will be loaded dynamically when the first Google Maps is set to visible. This approach will save API load count for Google Maps Enterprise clients.

 <script type="text/javascript" src="http://serverapi.arcgisonline.com/jsapi/arcgis/?v=2.1"></script>
<script src="/path/to/gmapslayer_compiled.js" type="text/javascript"></script>

Add a Google Maps Layer

After the script is loaded, you can simply treat Google Maps as other layer in your application. 

var gMap = new gmaps.GoogleMapsLayer({visible:false, id:'googlemaps'});
map.addLayer(gMap);

That's it! You should only need one instance of the layer, and use setMapType(mapTypeId) to switch between Google Street Map, Satellite, Hybrid and Terrain.
